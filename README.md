# МП Здравие — сайт с PHP-админкой и MySQL

## Что внутри

- `index.php` — главная страница для PHP-хостинга.
- `index.html` — статическая копия для GitHub Pages.
- `css/style.css` — стили.
- `js/app.js` — каталог, корзина, заявки, админка и обращение к PHP API.
- `api/` — PHP API для каталога, авторизации, настроек и товаров.
- `api/config.php` — настройки подключения к MySQL.
- `database/schema.sql` — SQL-запрос для phpMyAdmin: база, таблицы, индексы и стартовый админ.
- `data/products.json` — стартовый прайс. PHP перенесёт его в MySQL при первом запуске, если таблица `products` пустая.

## Установка на хостинге

1. Откройте phpMyAdmin.
2. Выполните SQL из `database/schema.sql`.
3. В `api/config.php` укажите данные MySQL:

```php
const DB_HOST = '127.0.0.1';
const DB_NAME = 'zdravie_db';
const DB_USER = 'ваш_пользователь';
const DB_PASS = 'ваш_пароль';
```

4. Загрузите проект на PHP-хостинг.
5. Откройте сайт через `index.php`.

## GitHub Pages

GitHub Pages не запускает PHP и MySQL. Он может показать только статическую версию сайта через `index.html`.

На GitHub Pages каталог загрузится из `data/products.json`, но админка с базой данных работать не будет. Для админки нужен обычный PHP-хостинг с MySQL/phpMyAdmin.

## Админка

Ссылка находится внизу сайта: `Управление прайсом`.

Стартовый вход:

- логин: `admin`
- пароль: `admin123`

После первого входа смените пароль во вкладке `Настройки`.

## Как теперь работают товары

Карточки/строки товаров берутся из таблицы `products` в MySQL. Добавление, редактирование и удаление в админке сразу сохраняются в базе. У открытых посетителей каталог автоматически обновляется каждые 5 секунд.

## SQL

Основной запрос для создания базы и таблиц лежит здесь:

```text
database/schema.sql
```
